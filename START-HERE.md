جهّزت لك التطبيق بالكامل: مفاتيح Supabase مدمجة، رمز سري قوي، المزامنة التلقائية مفعّلة، والنموذج اليدوي مخفي عن التاجر. يبقى **3 أشياء فقط** تقوم بها أنت.

---

## 1️⃣ شغّل قاعدة البيانات (مرة واحدة — 3 دقائق)

1. ادخل [supabase.com](https://supabase.com) → مشروعك.
2. من القائمة: **SQL Editor** → **New query**.
3. افتح ملف `supabase-schema.sql` (المرفق)، انسخ كل محتواه، والصقه.
4. اضغط **Run** (أو Ctrl+Enter).
5. يجب أن ترى ✓ Success.

> بدون هذه الخطوة، المزامنة لن تعمل (ستظهر أخطاء اتصال).

---

## 2️⃣ ارفع التطبيق على GitHub Pages (مرة واحدة)

1. أنشئ مستودعاً (repo) جديداً على GitHub.
2. ارفع محتوى **`mobile-pos-GITHUB.zip`** (فك الضغط أولاً).
3. **Settings → Pages** → Source: **GitHub Actions**.
4. انتظر دقيقتين. رابطك:
   ```
   https://USERNAME.github.io/mobile-pos/
   ```

> ⚠ لا ترفع `activation-tool.html` أبداً (فيه الرمز السري). حزمة GitHub خالية منه أصلاً.
> ✅ بعد الرفع على HTTPS: **الكاميرا ستعمل** ومسح الباركود سيشتغل.

---

## 3️⃣ (موصى به) جدّد مفتاح Supabase

لأنك نشرت المفتاح في المحادثة:
- **Settings → API → Reset anon key**.
- ضع الجديد في `index.html` مكان `SUPABASE_ANON`.
- ارفع النسخة المحدّثة.

## 🛒 كيف تبيع لتاجر (العملية الكاملة)

```
1. التاجر يفتح رابط التطبيق → 7 أيام تجربة مجانية
2. يفتح: الإعدادات → الترخيص → ينسخ "معرّف التثبيت"
3. يرسله لك عبر WhatsApp
4. أنت تفتح activation-tool.html (على جهازك فقط) → تلصق المعرّف → توليد كود
5. ترسل الكود للتاجر
6. التاجر يُدخل الكود → التطبيق يُفعّل + المزامنة تشتغل تلقائياً ✓
```

**جهاز ثانٍ لنفس التاجر:** يُدخل **نفس الكود** ويؤشّر خانة "هذا جهاز ثانٍ".
